# US-005: Client-Side Form Validation

**As a** customer
**I want** immediate feedback on my form input
**So that** I can correct errors before submitting

## Problem Statement

Users need immediate feedback when entering invalid data in the checkout form. Client-side validation provides instant feedback and improves user experience by catching errors before submission.

## Requirements

1. Validate email format (RFC 5322 compliant)
2. Validate phone number format (US format: XXX-XXX-XXXX or (XXX) XXX-XXXX)
3. Validate required fields are not empty
4. Validate address is not empty and has minimum length
5. Show inline error messages next to invalid fields
6. Clear error messages when field becomes valid
7. Prevent form submission if validation fails
8. Validate on blur and on submit

## Acceptance Criteria

1. **Given** I am filling out the checkout form **When** I enter an invalid email (e.g., "notanemail")
   a. **Then** I see an error message below the email field
   b. **And** the error message clears when I enter a valid email

2. **Given** I am filling out the checkout form **When** I enter a phone number in wrong format
   a. **Then** I see an error message below the phone field
   b. **And** the error message suggests the correct format

3. **Given** I have invalid fields in the form **When** I click submit
   a. **Then** the form does not submit
   b. **And** I see error messages for all invalid fields
   c. **And** the first invalid field is focused

## Definition of Done

- Code meets project standards
- Peer reviewed and approved
- Documentation updated
- All validation rules tested
- Error messages are clear and helpful
- Accessibility reviewed (ARIA labels for errors)
